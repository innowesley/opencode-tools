# Plan: Improve `install-plan-it.sh` instructions to prevent missing the plan-file save step

## Goal
Update the three instruction files generated by `install-plan-it.sh` so the Plan-mode instructions make saving the `.md` plan file to `.agents/plans/pending/` an explicit, unskippable two-step checklist — preventing the kind of miss where the plan is output in chat but never saved to disk.

## Root Cause
The current phrasing "then save it" is a trailing clause after "output the plan as formatted markdown in your response." It reads as optional/a secondary thought, making it easy to skip in practice. The fix is to elevate this to a **bold, numbered checklist** that must be completed in every Plan-mode response.

## Files to Modify (all generated by `install-plan-it.sh`)

| File | Change |
|------|--------|
| `opencode.json` (plan prompt) | Replace trailing "then save it" with bold two-step checklist + "MUST happen together" |
| `AGENTS.md` (Plan-First Workflow section) | Same treatment — explicit checklist with `[ ]` boxes |
| `skills/plan-flow/SKILL.md` (step 3 of "How to use") | Add a verification checklist after step 3 |

## Detailed Changes

### 1. `opencode.json` — plan prompt (line ~26 and ~306 and ~440 and ~573)
Current:
```
After writing the plan as formatted markdown in your response, save it to `.agents/plans/pending/<name>.md` using `bash cat << 'EOF' > <path>`. Do NOT use `write-plan` for new plans — its `content` argument displays the full plan in the UI as raw tool arguments, then use the `question` tool to ask...
```
New:
```
**CRITICAL — Plan requires TWO actions in SAME response:**
1. Output the full plan as **formatted markdown** in your response (main messages panel)
2. Save it to `.agents/plans/pending/<name>.md` using `bash cat << 'EOF' > .agents/plans/pending/<name>.md`
These two MUST happen together. NEVER output a plan without immediately saving it to file.

Do NOT use `write-plan` for new plans — its `content` argument displays the full plan in the UI as raw tool arguments, then use the `question` tool to ask...
```

### 2. `AGENTS.md` — Plan mode section (line ~54 and ~333 and ~467 and ~601)
Current:
```
- Output the plan as formatted markdown in your response (main messages panel), then save it with `bash cat << 'EOF' > .agents/plans/pending/<name>.md`
```
New:
```
1. **Output** the plan as formatted markdown in your response (main messages panel)
2. **Save** it to `.agents/plans/pending/<name>.md` using `bash cat << 'EOF' > .agents/plans/pending/<name>.md`
   → These two happen in the same response. Never do step 1 without step 2.
```

### 3. `skills/plan-flow/SKILL.md` — step 3 (line ~258 and ~528)
Add a verification checklist after step 3:
```
3. **Write plan** — Say "Writing plan..." then output the full plan content as **formatted markdown** in your response — it renders in the **main messages panel (center)** with native colors (bold, headings, code blocks). This is the primary way the user sees the plan. Do NOT use `read` on the `.md` file (shows ugly line numbers). Then save the plan file using `bash cat << 'EOF' > .agents/plans/pending/<name>.md` with the plan content as the heredoc.

   **Verification checklist (MUST complete before proceeding):**
   - [ ] Output plan as formatted markdown in main response
   - [ ] Save to `.agents/plans/pending/<name>.md` via heredoc
   - [ ] File was created (confirmed by lack of error from bash)
   - Missing either? You have a bug — fix it before using `question` tool.

   Do NOT use `write-plan` for new plans — its `content` argument displays the full plan in the UI as raw tool arguments, duplicating what should only be in the main messages panel.
```
